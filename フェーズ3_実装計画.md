# パフォーマンス最適化およびPPTX解析耐性強化の実装計画

ユーザーからの要望に基づき、「辞書データの最適化（自社配信化）」および「複雑なPPTXレイアウト時のテキスト抽出精度の向上」を行うフェーズ3の実装計画を策定しました。

## 実装を予定している変更内容

### 1. 辞書データの最適化（速度改善・安定化）
外部CDNへの依存をなくし、アプリ自身で辞書ファイル（約17MB）をホスティングするように変更します。これにより、以下のメリットが生まれます。
- **通信の高速化**: 外部ドメインへアクセスするオーバーヘッドが消滅。
- **可用性の向上**: CDN側がダウンしても自前のアプリは稼働し続ける。

**【具体的な対応】**
- 開発・ビルド時に `node_modules/kuromoji/dict` フォルダを静的領域（`public/` 相当）へ自動コピーする Vite用プラグイン `vite-plugin-static-copy` を導入。
- `kuroshiro` の辞書読み込みパスを `/dict/` （アプリのローカル相対パス）に変更。

### 2. 複雑なPPTXレイアウトへの耐性強化
現在はスライド内の `<a:t>`（テキストノード）を順番に単純結合しているため、テキストボックスが分かれている場合などに文字が不自然にくっついたり、改行が無視されてしまう課題があります。

**【具体的な対応】**
- XMLのパースロジックを改良し、テキストノードの親である **`<a:p>`（段落）や `<a:br>`（改行）タグを解釈** するようにします。
- **HTML出力時**: 同じ段落は1つのブロックとし、改行箇所には `<br>` を挿入することで、元のスライドレイアウトにかなり忠実な読みやすいHTMLとして出力させます（`slideTextContent` 構築プロセスを抜本的に強化します）。

- プラグインを追加するため、作業後にターミナルから再度 `npm install` していただく必要があります。
